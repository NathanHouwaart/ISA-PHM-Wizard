# Add Configurations Feature to Test Setups and Studies

## Overview
Add a new "Configurations" tab to the Test Setup page where users can define test setup configurations. These configurations can then be selected and linked to studies in the Study slide.

## 1. Data Structure Changes

### Test Setup Object
Add a `configurations` array to each test setup object:
```javascript
{
  id: "...",
  name: "...",
  // ... existing fields ...
  configurations: [
    {
      id: "uuid",
      name: "Configuration A",
      details: [
        { id: "uuid", name: "Replaceable Component ID", value: "RC-001" },
        { id: "uuid", name: "Other Detail", value: "Some value" }
      ]
    }
  ]
}
```

### Study Object
Add a `configurationId` field to study objects:
```javascript
{
  id: "...",
  name: "...",
  // ... existing fields ...
  configurationId: "" // ID of selected configuration from the test setup
}
```

## 2. Test Setup Form Updates (`src/components/TestSetup/TestSetupForm.jsx`)

### Add Configurations Tab
- Add "configurations" tab to the main TabSwitcher (alongside "general", "sensors", "characteristics")
- Tab should appear as: `{ id: 'configurations', label: 'Configurations', tooltip: 'Manage test setup configurations' }`

### Create ConfigurationsEditor Component
Model after `SensorsEditor` and `CharacteristicsEditor` components:
- Collapsible cards for each configuration
- Each card shows: `c{index}: {name} - {detailsCount} details`
- Expandable to show:
  - Name field (text input)
  - Details section (flexible key-value pairs like sensors' additionalInfo)
  - Add/remove detail entries
  - Each detail entry has: name (label) and value fields
- Add Configuration button
- Remove Configuration button per item
- Help tooltip with examples

### Add Grid View for Configurations
- Create a second view toggle (Simple View / Grid View) within the Configurations tab
- Grid columns:
  - Actions column (delete row)
  - ID column (auto-generated, read-only, pattern: `Config C{nn}`)
  - Name column (editable)
  - Details Count column (read-only, shows count of detail entries)
  - Details Hint column (read-only, shows preview like "Replaceable Component ID: RC-001, ...")
- Grid config similar to `characteristicGridConfig` and `sensorGridConfig`

### Initial State
Add to `formData` initial state:
```javascript
configurations: item?.configurations || []
```

## 3. Study Form Updates (`src/components/Study/StudyForm.jsx`)

### Add Configuration Selection Field
- Add a FormField of type "select" (dropdown)
- Field name: `configurationId`
- Label: "Test Setup Configuration"
- Options populated from: `testSetups.find(t => t.id === selectedTestSetupId)?.configurations || []`
- Display: configuration name in dropdown
- Value: configuration id
- Include empty option: "No configuration selected"
- Explanation: "Select which configuration of the test setup was used for this study"
- Handle case where no test setup is selected (show disabled field with message)

### Access Global Context
Import and use:
```javascript
const { testSetups, selectedTestSetupId } = useGlobalDataContext();
```

### Initial State
Add to `formData`:
```javascript
configurationId: item?.configurationId || ''
```

## 5. Implementation Guidelines

### Follow Existing Patterns
- Configuration editor should mirror the structure of CharacteristicsEditor and SensorsEditor
- Use same UI components: FormField, IconTooltipButton, TableTooltip, etc.
- Use uuid4() for generating IDs
- Details array follows same pattern as sensors' additionalInfo

### State Management
- All data flows through GlobalDataContext
- Configurations stored within test setup objects (not separate global state)
- Studies store only the configurationId reference

### Validation
- Configuration name is required when saving test setup
- No validation needed for configurationId in studies (can be empty)

### Default Values
- New test setups: `configurations: []`
- New configurations: `{ id: uuid4(), name: '', details: [] }`
- New detail entries: `{ id: uuid4(), name: '', value: '' }`

### Grid View Implementation
- Both tabs (configurations in test setup, studies in slide) need dropdown support
- For StudySlide grid: configuration dropdown should filter to only show configurations from the currently selected test setup
- Handle edge case: if selectedTestSetupId is null/undefined, disable or hide configuration column
